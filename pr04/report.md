## 2. auditd — настройка правил
--------------------------------------------------------------------------------

### Применённые правила (вывод auditctl -l)

### Разбор записи SYSCALL
type=SYSCALL msg=audit(05/07/2026 12:49:09.609:359) : arch=x86_64 syscall=openat success=yes exit=5 a0=AT_FDCWD a1=0x7ff52fe6d62a a2=O_RDONLY|O_NONBLOCK a3=0x0 items=1 ppid=2760 pid=2935 auid=kotori uid=root gid=kotori euid=daemon suid=root fsuid=daemon egid=root sgid=kotori fsgid=root tty=pts0 ses=3 comm=sudo exe=/usr/bin/sudo subj=unconfined key=watch-sudoers


| Поле | Значение | Что означает |
|------|---------|-------------|
| auid | kotori | исходный пользователь |
| uid | root | реальный пользователь |
| euid | daemon | эффективный пользователь |
| comm | sudo | имя команды |
| exe | /usr/bin/sudo | путь к файлу |

### Как auditd отличает чтение файла от его изменения
| Действие | syscall | Флаги открытия |
|----------|---------|----------------|
| Чтение | openat | O_RDONLY |
| Изменение | openat | O_WRONLY / O_RDWR |

## 3. Расследование инцидента
--------------------------------------------------------------------------------

### Хронология действий
| Время | Действие | Результат | Источник |
|-------|---------|-----------|----------|
| 12:49:09 | ausearch -k watch-sudoers -i | Успех | ausearch -k watch-sudoers |

### Как auditd помог расследованию
- Определил, кто выполнял команду (auid=bob)
- Зафиксировал права доступа (uid=root, euid=daemon)

### Чем auditd лучше journalctl для задач аудита безопасности
- Фиксирует системные вызовы на уровне ядра
- Сохраняет неизменный auid
- Отслеживает доступ к конкретным файлам
