# ПР №9. Следы вредоносного ПО в Linux

## 1. Что было посажено

| Механизм | Место | Команда/файл |

| Cron | crontab пользователя | /tmp/.hidden_malware/backdoor.sh |

## 2. Что нашли — процессы

**Команда:** `ps aux`

**Вывод:**

USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND

root 3689 0.0 0.1 6940 3260 pts/0 S 15:40 0:00 /bin/bash /tmp/.hidden_malware/listener.sh

root 3690 0.0 0.1 6940 3260 pts/0 S 15:40 0:00 /bin/bash /tmp/.hidden_malware/listener.sh

**Что подозрительно:**

- Запуск из `/tmp/` — временная директория не должна содержать исполняемые скрипты

- Скрытая директория `.hidden_malware` — название указывает на вредоносное ПО

- Запуск от root — скрипт работает с максимальными привилегиями

- Два экземпляра `listener.sh` — возможно, дублирование для надёжности

- PID 3689 и 3690 — запущены с небольшим интервалом

## 3. Что нашли — сетевые соединения

**Команда:** `ss -tulnp`

**Вывод:**

tcp LISTEN 0 1 0.0.0.0:4444 0.0.0.0:* users:("nc",pid=3690,fd=3)

tcp LISTEN 0 1 0.0.0.0:4444 0.0.0.0:* users:("nc",pid=3635,fd=3)

**Подозрительный порт:** 4444

**Процесс:** `nc` (netcat) с PID 3635 и 3690

**Команда:** `sudo lsof -i :4444`

**Вывод:**

COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME

nc 3635 root 3u IPv4 43200 0t0 TCP *:4444 (LISTEN)

nc 3690 root 3u IPv4 43323 0t0 TCP *:4444 (LISTEN)

**Как lsof связывает порт с процессом:**

1. `lsof -i :4444` показывает все открытые файловые дескрипторы, связанные с портом 4444

2. Каждый сетевой сокет в Linux — это файловый дескриптор (FD)

3. В выводе видно: COMMAND (nc), PID (3635, 3690), USER (root), FD (3u — файловый дескриптор №3), NAME (TCP *:4444 (LISTEN))

4. `nc` (netcat) — классический инструмент для создания бэкдоров, слушает входящие соединения на всех интерфейсах

## 4. Что нашли — автозапуск

### Cron

**Команда:** `sudo cat /var/spool/cron/crontabs/*`

**Вывод:**

```bash

@reboot /tmp/.hidden_malware/backdoor.sh &

*/5 * * * * /tmp/.hidden_malware/backdoor.sh &

```

**Что подозрительно:**

- `@reboot` — запускается при каждой перезагрузке системы

- `*/5 * * * *` — запускается каждые 5 минут, постоянно перезапуская вредонос

- Путь `/tmp/.hidden_malware/backdoor.sh` — указывает на скрытую директорию во временной папке

- Две записи гарантируют, что даже если одна не сработает, вторая выполнится

### Systemd

**Команда:** `systemctl --user list-unit-files | grep system-helper`

**Содержимое unit-файла:**

```ini

[Unit]

Description=System Helper Service

After=default.target

[Service]

ExecStart=/tmp/.hidden_malware/backdoor.sh

Restart=always

RestartSec=10

[Install]

WantedBy=default.target

```

**Что подозрительно:**

- `Restart=always` — сервис автоматически перезапускается при любом завершении

- `RestartSec=10` — перезапуск через 10 секунд, делает его "живучим"

- Название "System Helper" — маскировка под легитимный системный сервис

- Путь к `backdoor.sh` из `/tmp/` — временная директория для системного сервиса

- Юнит в `~/.config/systemd/user/` — скрыт от обычного `systemctl list-unit-files` без `--user`

### ~/.bashrc

**Команда:** `grep -n "backdoor" ~/.bashrc`

**Строка которую нашли:**

```bash

/tmp/.hidden_malware/backdoor.sh &

```

**Где находится:** конец файла ~/.bashrc (последние строки)

**Что подозрительно:**

- Добавление в `~/.bashrc` обеспечивает запуск при каждом входе в систему

- Запускается в фоне (`&`), не мешая работе пользователя

- Использует тот же путь `/tmp/.hidden_malware/backdoor.sh`

- Тройной механизм автозапуска (cron + systemd + bashrc) гарантирует постоянную активность

## Выводы

В ходе выполнения лабораторной работы были найдены множественные следы вредоносного ПО в системе:

- Основной вредонос — скрипты `listener.sh` и `backdoor.sh`, размещённые в скрытой директории `/tmp/.hidden_malware/`

- Бэкдор — через `netcat` (nc) открыт порт 4444 для внешнего доступа

- Тройная система автозапуска:

  - Cron — запуск при перезагрузке и каждые 5 минут

  - Systemd — постоянный перезапуск через 10 секунд

  - `~/.bashrc` — запуск при входе пользователя

Самое неочевидное место обнаружения вредоноса — это `~/.config/systemd/user/system-helper.service`, потому что:

- Большинство администраторов проверяют только системные юниты (`/etc/systemd/system/`)

- Пользовательские юниты скрыты от `systemctl list-unit-files` без флага `--user`

- Название "System Helper Service" маскируется под легитимный системный процесс

- `Restart=always` делает его практически неубиваемым

**Важность комплексной проверки** — только совместный анализ всех механизмов (процессы, сеть, автозагрузка) позволяет полностью выявить вредоносное ПО. Проверка только одного механизма оставила бы злоумышленнику возможность восстановить контроль над системой.
