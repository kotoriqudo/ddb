Изменение системного времени |

| SYS_RAWIO | Доступ к портам ввода-вывода |

| MAC_OVERRIDE | Обход MAC-ограничений |

| SYS_BOOT | Перезагрузка системы |

| SYS_NICE | Изменение приоритетов процессов |

**Опасность --privileged:**

- Даёт полный доступ к хосту

- Контейнер может "сбежать" на хост

- Можно монтировать корневую файловую систему хоста

- Можно изменять настройки ядра

- Можно перехватывать трафик

- Можно добавлять сетевые интерфейсы

## 4. Эшелонированная защита

| Слой | Инструмент | Что ограничивает |

|------|------------|------------------|

| DAC | chmod/chown | Права на файлы (чтение/запись/выполнение) |

| Capabilities | --cap-drop ALL + cap-add | Только необходимые права для процессов |

| MAC | AppArmor | Доступ к файлам и системным вызовам |

| Изоляция | Docker namespaces | Процессы, сеть, файловая система |

## Выводы

- **Capabilities** позволяют давать программам минимальные права без полного root-доступа.

- **AppArmor** работает как дополнительный слой безопасности (MAC) поверх стандартных прав (DAC).

- **Docker-контейнеры** по умолчанию имеют ограниченный набор capabilities для безопасности.

- **--privileged** даёт практически полный доступ к хосту — использовать только в крайних случаях.

- **Эшелонированная защита** (DAC → Capabilities → AppArmor → Namespaces) создаёт несколько барьеров для злоумышленника.

- **Практика минимальных привилегий** — ключевой принцип безопасности.

- **Монтирование конкретных папок** безопаснее, чем `--privileged`, так как ограничивает доступ контейнера к файловой системе хоста.

- **Запуск контейнеров от root** (через сокет Docker) — потенциальная уязвимость, поэтому важно использовать `--user` и не давать доступ к Docker непривилегированным пользователям.
